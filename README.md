# SYSU-Food-Picker (康乐园美食审判台)

> **项目初衷**：今晚吃什么，这是一件大事！本项目初版旨在通过简单的随机算法，帮助中山大学康乐园（南校园）的同学们快速决定下一餐，解决每天由于选择过多导致的“决策疲劳”。

---

## 🌟 核心功能回顾 (V1.0 基础版)

在最初的设计中，我们构建了决策系统的雏形，实现了以下基础能力：
* **分类均衡随机**：确保四大分类（食堂、外卖、出去吃、泡面）概率相等，不偏袒任何一种干饭方式。
* **加权推荐算法**：引入评分机制，按 5:4:3:2:1 比例加权，确保高评价的店有更高概率被选中。
* **中大南校定制**：预设了松涛园、下渡路等周边真实美食数据，贴合实际生活场景。

---

## 🚀 重大版本更新报告 (V2.0 智能决策版)

在最新的 2.0 迭代中，我们对决策引擎进行了深度的“智能化”重构，使其从一个简单的随机工具进化为具备“环境感知”能力的干饭助手。

### 1. 🌍 环境感知与时空算法 (Context-Awareness)
* **设计目的**：让 App 拥有“常识”。
* **功能实现**：
    * **天气补偿**：新增“雨天模式”。下雨时会自动降低“出去吃”的概率，并提升“点外卖”与“吃泡面”的权重。
    * **时空校准**：自动读取系统时间。针对 18:00 等食堂高峰期自动修正排队时长预测，而在 15:00 等闲时段自动忽略火爆度惩罚。

### 2. ⏳ 智能等待预测与递减机制
* **设计目的**：在效率与惊喜之间寻找平衡。
* **功能实现**：
    * **常识逻辑优化**：泡面固定 3-5 分钟（物理浸泡时间），外卖锁定在 25-50 分钟合理区间，拒绝离谱预测。
    * **负反馈递减**：若用户因嫌等太久而点击“重新选择”，系统会记忆上次时值，确保下一轮生成的预测时长严格递减。

### 3. 💰 多维意图驱动与财务约束
* **设计目的**：根据用户当下的实际“贫富”状态和时间状态进行决策。
* **功能实现**：
    * **四象限偏好**：提供“综合考量、极限省钱、效率至上、美味优先”模式，动态重构底层权重。
    * **硬性预算拦截**：引入渐进式预算滑块，开启后将自动封印超出预算的店铺，实现财务自律。

### 4. 📂 工业级数据录入与健壮性
* **设计目的**：解决输入低效问题，保证系统稳定。
* **功能实现**：
    * **智能批量解析**：支持多行文本一键导入，通过正则匹配自动提取名称、分类与预算。
    * **防御性编程**：内置数据迁移保护，自动识别并修复旧版本残留的“脏数据”，防止应用崩溃。

## 🛠️ 技术栈与架构

* **开发语言**: ArkTS (HarmonyOS 声明式开发语言)
* **UI 框架**: ArkUI (内置毛玻璃特效与物理弹簧动画)
* **持久化层**: PersistentStorage (确保个人饭单永不丢失)
* **核心逻辑**: 多目标优化决策算法 (Multi-objective Optimization)
